We have completed the Wodify capability discovery + raw staging foundation. Now I want to build the next architectural layer: canonical mapping and promotion rules.

Context:
- We are intentionally moving Iron Metrics toward a connector/staging/canonical architecture.
- Long term, this should grow into a full gym operating system.
- This sprint should NOT attempt a total product rewrite.
- It should introduce the first canonical data layer on top of the staged Wodify data.
- The goal is to stop relying on direct Wodify-shaped imports for core analytics.

GOAL OF THIS SPRINT:
Using staged Wodify payloads and the saved source profile, Iron Metrics should be able to:
- map staged source records into canonical candidate records
- persist canonical candidate records with confidence/completeness metadata
- apply promotion rules to determine which candidates are safe for analytics-ready use
- produce clear diagnostics on why records were promoted, staged, or blocked

This sprint should focus only on 3 core domains:
1. people
2. memberships
3. attendance

Do NOT attempt a full billing/revenue canonicalization yet unless it is trivial to scaffold.

==================================================
5. ADD PROMOTION RULES
==================================================

Build a promotion step that decides whether a canonical candidate is safe to be considered analytics-ready.

Rules:
- A person can be promoted if there is a reliable identity basis
- A membership can be promoted if it has a reliable membership identifier and enough date/status linkage, even if some optional fields are missing
- Attendance can be promoted if it has a valid attendance date and enough person linkage or stable external identity reference

If a record is not safe to promote:
- keep it as blocked or candidate
- preserve validation reasons
- do not silently discard it

This sprint does NOT need to rewrite the whole analytics engine to consume these tables yet.
But these promoted canonical records should be clearly marked as the future analytics-ready layer.
