# 1️⃣ Replace Stub Generator with Real LLM Call

## Backend Deliverables

Create:

`/services/operator/generator.ts`

This service must:

* Accept:

  * gym context (aggregated metrics only)
  * selected pillar
  * task type
* Build a structured system prompt
* Call LLM provider
* Validate response against schema
* Retry once if schema invalid
* Log structured output to DB

### CRITICAL RULE

The model must be forced to return JSON only.

No prose.
No markdown.
No explanation outside schema.

---

# 2️⃣ Output Quality Rules (Non-negotiable)

These are enforced both in prompt and server validation:

### Length Rules

* `why_it_matters` ≤ 240 characters
* `actions` must be 4–7 items
* No action longer than 140 characters
* Draft messages ≤ 600 characters

### Tone Rules

* Calm
* Direct
* No hype
* No emojis
* No motivational fluff
* No “game changing” language
* No guarantees

### Prohibited Language

* “This will guarantee…”
* “Legally…”
* “Tax strategy…”
* “Medical…”
* “You should consult a lawyer…” (unless user explicitly requests)

# 5️⃣ Schema Enforcement Layer

Use runtime validation (Zod or equivalent).

If invalid:

* Retry with “Return valid JSON only”
* If still invalid:

  * Log error
  * Return controlled fallback output

This ensures:
No broken UI.
No partial text blobs.

This layer is simple but critical.
